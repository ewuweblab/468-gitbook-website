# Physical Web

## How We Got Here

> The core vision — a URL on everything, no app required — is still being pursued through these browser APIs.

{% embed url="https://developers.google.com/web/shows/google-io/2015/physical-web" %}

An effort by Google to extend the URL — the web's core primitive — to everyday physical objects. The premise: walk up to any smart object (vending machine, poster, bus stop, rental car) and interact with it **without installing an app first**. Just tap and use.

**Real-world examples**

* A **bus** alerting riders of the next stop
* A **city bike-share** letting users sign up on the spot
* A **home appliance** offering an interactive tutorial
* An **industrial robot** displaying diagnostic info
* A **mall** offering a map
* **Restaurant menus** with info on specials, food provenance, and wait times
* A **lost dog's collar** beacon to help reunite it with its owner [DeviceAtlas](https://deviceatlas.com/blog/physical-web-dead-long-live-physical-web)

**Tech**

* **Web NFC** (tap-based URL interaction, no BLE needed)
* **Web Bluetooth** (direct BLE interaction from the browser)
* Broader **IoT + Web** convergence (PWA + device APIs)
